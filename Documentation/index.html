Equestria OS Packages

Набор системных пакетов для Equestria OS: автоматическое управление NVIDIA-драйверами, поддержание дистро-идентичности, GPG-инфраструктура доверия, уведомления об обновлениях и перезагрузках.

NVIDIA DKMS GPG Keyring GPU Sync Pacman Hooks Initramfs Reboot Notify Update Checker Arch Linux

📦Обзор пакетов

Семь взаимосвязанных пакетов образуют инфраструктуру Equestria OS.

🔴 equestria-nvidia-dkms

Самообновляемые проприетарные NVIDIA-модули ядра через DKMS. Для legacy GPU (Maxwell / Pascal / Volta).

🟣 equestria-os-keyring

GPG-связка ключей для верификации пакетов Equestria OS. RSA 4096-bit, интеграция с pacman-key.

🔵 pg-gpu-sync

Автодетекция GPU и выбор оптимального драйвера при загрузке. Поддерживает открытые и проприетарные драйверы.

🟢 pg-hooks

Сохранение брендинга Equestria OS после обновлений filesystem и lsb-release. Управляет os-release и issue.

🟠 pg-nvidia-hook

Автоматическая пересборка initramfs при любом изменении NVIDIA-драйверов. Проверяет состояние DKMS.

🟡 pg-reboot-notify

Обнаружение необходимости перезагрузки после обновления ядра или systemd. KDE-уведомление пользователю.

🩵 pg-update

Почасовая проверка обновлений из pacman, AUR, Flatpak, Snap. KDE-уведомление при наличии новых версий.

📁Структура пакета

Каждый пакет: PKGBUILD + основной скрипт(ы) + .hook файлы + .install хук. Версии в формате MAJOR.MINOR.PATCH или YYYYMMDD.

🪝Libalpm hooks

Все автоматические реакции на события pacman реализованы через хуки в /usr/share/libalpm/hooks/*.hook. Тип: PostTransaction.

🔔Уведомления

Используется notify-send (libnotify). App name всегда "Equestria OS". Уведомления совместимы с KDE Plasma.

🌐Дистро-идентичность

Prefix pg- = Psyche Games. equestria- = дистро-специфика. URL: https://psyche-games.com.

📜Лицензии

GPL3 для всех системных пакетов. LicenseRef-NVIDIA-proprietary для nvidia-dkms. GPL для keyring.

🔧Оболочка

Все скрипты — Bash (#!/bin/bash). Пути исполняемых файлов: /usr/bin/. Вспомогательные скрипты: /usr/lib/<pkgname>/.

equestria-os-keyring

v20260407-1 · any · GPL

GPG-инфраструктура доверия для пакетов Equestria OS. Предоставляет связку открытых ключей для верификации подписей пакетов через pacman. После установки автоматически интегрирует ключ в локальный keyring системы.

equestria-os-trusted/usr/share/pacman/keyrings/Список доверенных отпечатков. Содержит fingerprint: 24B88A3AFD6B4E71F2C7EE44DA7ED220DA39685A:

Install hook

equestria-os-keyring.install
post_install() {
    # Добавить ключ в keyring пользователя pacman
    pacman-key --add /usr/share/pacman/keyrings/equestria-os.gpg

    # Установить доверие согласно equestria-os-trusted
    pacman-key --populate equestria-os
}
post_upgrade() { post_install; }
ℹ️
Команда pacman-key --populate equestria-os читает файл /usr/share/pacman/keyrings/equestria-os-trusted и подписывает перечисленные ключи локально с уровнем доверия "полное" (marginal).

pg-nvidia-hook

v1.0.4-1 · any · GPL3 · replaces: eos-nvidia-fix

Автоматическая пересборка initramfs при любом изменении NVIDIA-драйверов. Перед пересборкой проверяет состояние DKMS — если модуль не скомпилирован, пересборка пропускается с предупреждением, чтобы не сохранить сломанное состояние. Поддерживает mkinitcpio, dracut и booster.

DkmsModuleReady() — проверка DKMS
function DkmsModuleReady() {
    # Если dkms не установлен — всегда готов (pre-built драйвер)
    command -v dkms &>/dev/null || return 0

    # Проверяет: dkms status | grep "^nvidia.*: installed"
    dkms status | grep -qP '^nvidia[^,]*.+: installed'
    return $?
}
🔴
Важно: Если DKMS не скомпилировал модуль (например, нет kernel-headers), пересборка initramfs пропускается намеренно. Иначе система загрузится без NVIDIA и возможно без графики. Вместо этого выводится:
Install matching kernel headers, then: sudo dkms autoinstall && sudo mkinitcpio -P

🟡pg-reboot-notify

pg-reboot-notify

v1.0.0-1 · any · GPL3

Определяет необходимость перезагрузки после обновления ядра или systemd. Сравнивает запущенное ядро с установленным, проверяет флаги reboot-required. Если нужна перезагрузка — отправляет KDE-уведомление на русском языке.

libnotify bash notify-send KDE Plasma

Методы определения необходимости перезагрузки

RebootNeededKernel()

Сравнивает uname -r (текущее ядро) с ALL_kver= из /etc/mkinitcpio.d/*.preset. Если есть расхождение — нужна перезагрузка.

RebootNeededSystemd()

Проверяет systemctl is-system-running. При статусах degraded или starting возвращает true. (Определена, но не используется в Main)

RebootNeededFlag()

Проверяет наличие файлов /var/run/reboot-required или /run/reboot-required. Стандартный механизм Debian-систем.
